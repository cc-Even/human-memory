# Multi-Agent Memory Management System

A memory management system based on a multi-agent architecture that can automatically ingest, consolidate, and retrieve information.

---

## 🏗️ System Architecture

### Three-Layer Architecture

```
┌─────────────────────────────────────┐
│  User Interaction Layer (CLI + File Upload) │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  Memory Orchestrator (RootAgent)     │
│  ┌────────┬─────────┬──────────┐    │
│  │Ingest  │Consolidate│ Query   │    │
│  │Agent   │Agent      │ Agent   │    │
│  └────────┴─────────┴──────────┘    │
│  ┌──────────────────────────────┐    │
│  │ ConsolidationScheduler       │    │
│  │ (Auto Scheduled Consolidation)│    │
│  └──────────────────────────────┘    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  SQLite DB (memory.db)               │
└─────────────────────────────────────┘
```

### Core Components

#### 1. User Interaction Layer
- **Function**: Provides a command-line chat interface and file upload functionality.
- **Interaction**: Communicates directly with the Memory Orchestrator.

#### 2. Task Processing Layer

**Memory Orchestrator (RootAgent)**
- **Role**: The "brain" or central dispatch hub.
- **Responsibilities**: Receives instructions, coordinates sub-agents, schedules workflows, manages scheduler.

**IngestAgent**
- Summarize content.
- Extract key entities.
- Tag topics for the content.
- **Assess content importance (0-1 score)**.
- Store the results in the database after embedding.

**ConsolidateAgent**
- Find patterns and regularities in information.
- Merge related memory fragments.
- Associate memories through vector retrieval.
- Build connections between memories.

**ConsolidationScheduler**
- Periodically executes memory consolidation tasks.
- Configurable time interval and enable/disable switch.

**QueryAgent**
- Search for memories in the memory bank.
- Synthesize answers based on retrieved information.

#### 3. Data Storage Layer
- **Database**: SQLite (memory.db)
- **Function**: Persistently stores all memory data including vectors, entities, topics, and importance scores.

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure Environment Variables

#### Selecting LLM Provider

Set `DEFAULT_LLM_PROVIDER` in the `.env` file:

```env
DEFAULT_LLM_PROVIDER=dashscope  # or gemini openai
```

#### Copy the environment variable template and fill in your API keys

```bash
cp .env.example .env
```

Edit the `.env` file and enter your API keys:

```env
DASHSCOPE_API_KEY=your_dashscope_api_key_here
GEMINI_API_KEY=your_gemini_api_key_here
OPENAI_API_KEY=your_openai_api_key_here
```

### 3. Initialize Database

Before running the system for the first time, it is recommended to initialize the database:

```bash
python scripts/init_db.py
```

This script will create the necessary tables and ask if you want to generate sample data.

### 4. Run the System

```bash
python -m memory_system.ui.cli
```

## ⚙️ Configuration Parameters

### Vector Search Configuration

```env
VECTOR_SEARCH_TOP_K=5          # Number of top similar results to return
SIMILARITY_THRESHOLD=0.7       # Similarity threshold
```

### QueryAgent Configuration

```env
QUERY_TOP_K=5                      # Number of memories to return
QUERY_RELEVANCE_THRESHOLD=0.3      # Relevance threshold
QUERY_MAX_CONTEXT_LENGTH=4000     # Maximum context length
QUERY_ENABLE_VECTOR_SEARCH=true    # Enable vector search
QUERY_ENABLE_CONVERSATION=true     # Enable multi-turn conversation
QUERY_TEMPERATURE=0.7              # Temperature parameter
QUERY_MAX_TOKENS=1000              # Maximum tokens
```

### ConsolidateAgent Configuration

```env
CONSILIDATE_SIMILARITY_THRESHOLD=0.75   # Memory relation threshold
CONSILIDATE_TOP_K=5                     # Number of related memories to search
CONSILIDATE_MAX_MERGE_COUNT=3           # Maximum memories to merge
CONSILIDATION_INTERVAL_HOURS=24          # Auto consolidation interval (hours)
AUTO_CONSOLIDATE_ENABLED=false          # Enable auto consolidation
```

## 📊 Memory Data Model

Each memory contains the following fields:

| Field | Description |
|-------|-------------|
| id | Unique identifier |
| content | Original content |
| summary | Content summary |
| entities | Extracted entity list (JSON) |
| topics | Topic tags list (JSON) |
| embedding | Vector representation |
| importance_score | Importance score (0-1) |
| source_type | Source type (user_input/file_upload/consolidation) |
| is_merged | Whether memory has been merged |
| created_at | Creation timestamp |
| updated_at | Update timestamp |
